On the network layer, decentralisation is evaluated based on the distribution of nodes across organisations. On the geography layer, it is based on the distribution of nodes across countries. The ledgers that are currently supported for these two layers are: <Chain ledger="bitcoin">Bitcoin</Chain>, <Chain ledger="bitcoin_cash">Bitcoin Cash</Chain>, <Chain ledger="cardano">Cardano</Chain>, <Chain ledger="ethereum">Ethereum</Chain>, <Chain ledger="litecoin">Litecoin</Chain>, <Chain ledger="dogecoin">Dogecoin</Chain> and <Chain ledger="zcash">Zcash</Chain>. The following metrics are presented in the dashboard: <Term>Herfindahl-Hirschman index (HHI)</Term>, <Term>Nakamoto coefficient</Term>, <Term>1-concentration ratio</Term> and <Term>Shannon entropy</Term> (only for geography).

The blockchain data that is needed to determine these distributions are collected using crawlers running on each network. The results are based only on data we have collected and do not include extensive historical data.

For the <Chain ledger="bitcoin">Bitcoin</Chain>, <Chain ledger="bitcoin_cash">Bitcoin Cash</Chain>, <Chain ledger="litecoin">Litecoin</Chain>, <Chain ledger="dogecoin">Dogecoin</Chain> and <Chain ledger="zcash">Zcash</Chain> networks, the crawler we use is based on the <AppLink href="https://github.com/ayeowch/bitnodes">Bitnodes</AppLink> project. More than half of the <Chain ledger="bitcoin">Bitcoin</Chain> nodes use Tor (see <AppLink href={`${networkTo}/contributor`} target="\_self">Organisation Distribution</AppLink> and <AppLink href={`${geographyTo}/contributor`} target="\_self">Country Distribution</AppLink>), and it is therefore impossible to know by which organisations they are hosted or in which countries they are located. For the metrics, it was therefore decided to distribute these nodes proportionally among the different organisations/countries.

In <Chain ledger="ethereum">Ethereum</Chain>'s case, the network architecture consists of two distinct layers: execution and consensus. The dashboard reflects this architectural separation. The crawler used is <AppLink href="https://github.com/cskiraly/fast-ethereum-crawler">fast-ethereum-crawler</AppLink>.

Finally, data about <Chain ledger="cardano">Cardano</Chain> relay nodes is collected using <AppLink href="https://blockfrost.io/">Blockfrost</AppLink>.
