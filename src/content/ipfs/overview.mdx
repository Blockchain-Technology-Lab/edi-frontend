Thank you for helping the **Edinburgh Blockchain EDI Dashboard**! This guide walks you through sharing a dataset with us using IPFS — a simple, free way to send large files without email attachments or cloud storage limits.

You do **not** need to be technical to follow this. Just follow the steps in order using the navigation below.

Many blockchain datasets are extremely large (multiple GBs or even TBs). Instead of uploading files through email or cloud storage, we recommend IPFS (InterPlanetary File System) because it provides:

- efficient sharing of large datasets
- content-addressed storage (every dataset has a unique CID)
- resumable downloads
- decentralized distribution
- easy verification that data has not changed

You only need to send us the **CID (Content Identifier)** after uploading your dataset. If another transfer mechanism is more appropriate for your infrastructure, please <AppLink href="mailto:edi@ed.ac.uk">contact us</AppLink> instead.
